今天（或者是說從昨天晚上睡前）一口氣看了好幾部關於 Macbook Neo 的影片
包括影視颶風、最速男鍾文澤還有 MKBHD 的影片，還有一個攝影相關的 ytr 在測試 Macbook Neo 是否真的不能執行重度 loading 的操作

> 不過 MKBHD 說這是他覺得過去十年最顛覆性的產品 (... not only is this really good, I think it's potentially Apple's most disruptive product in the last 10 plus years.)，比起 iPhone X 和 Vision Pro 更顛覆性
> 我自己是不這麼認為啦，雖然我也同意 iPhone X 和 Vision Pro 並不是過去十年最顛覆性的產品
> 不如說過去十年讓我最驚豔的應該只有蘋果幾乎無痛轉的 `將 Mac 從 x86 換到 Arm 架構上` 這件事，反而不是任何產品

先說結論：除非我未來的工作流有大幅度改變，否則我一定不會買，但我覺得這台會賣的很好

## CP 值之王

CP 值——或是說價格——應該是這台 Macbook 最大的優點了，蘋果這次用極度精準的刀法切出了這個產品的最佳定位
如果十年前有人跟我說只要 19990 就能買到最新代的 Macbook，我一定會說他瘋了
我只能說庫克真不愧是供應鏈出身的，許多部件我相信都是直接從倉庫裡拉出來（直接拿前一代 iPhone 的晶片）、或是現有供應鏈簡單調整一下就好（鍵盤設計基本上沿用目前 Macbook Air 使用的，只是少了背光 & 換顏色）

晶片選用 iPhone 16 Pro 的 A18 Pro，不過只有 6 顆 CPU 核心 + 5 顆 GPU 核心，推測少一顆 GPU 核心是要讓通道給 USB，同時 Ram 也是焊死的 8GB，完全沒辦法升級
這台筆電唯二可以選擇的東西是顏色和儲存空間，SSD 也只有 256GB & 512GB 可選，其中 512GB 版本多支援 TouchID
螢幕面板也是沿用 Macbook Air 的，不過只支援 sRGB 不支援 P3 廣色域
觸控板不是現在 Macbook Air/Pro 用的，退回純物理按鍵，上次看到這樣的設計還是 Macbook Air 2015 了
最後是這台筆電只有兩個 Type-C + 3.5mm 耳機孔，而且其中一個 Tpye-C 只有 USB 2.0、另一個是 USB 3.0，不支援 Thunderbolt

不過想一想也和合理，雖然說 Thunderbolt 是蘋果和 Intel 一起開發的標準、目前也捐給 USB 基金會了，但當時 iPhone 16 Pro 不支援 Thunderbolt，估計沒有整合在 A18 Pro 裡面
如果多支援就需要多一顆晶片、多一顆晶片價格勢必要上升，那沒有這功能好像也沒什麼問題，反正我是不認為 Thunderbolt 是鋼需求，我自己也只有在需要外接 PCIe SSD 的時候才有刻意找支援 Thunderbolt 的設備
至於只有一個 Type-C 支援 USB3.0 就比較煩躁一點了，我先猜應該是 A18 Pro 本身的限制，即便少一顆 GPU 核心多讓通道了還是不夠用

至於效能的部分，從跑分軟體上可以發現 A18 Pro 單核效能大概對標 M4、多核效能對標 M1
所以基本上可以說你在 M1 Mac 上面能做到的事 Macbook Neo 應該也是完全沒問題的

## 便宜治百病

雖然相比於這幾年的 Macbook 使用體驗，我必須說 Macbook Neo 一定是沒達到標準的
像是自從 2015 年以後蘋果再也沒有推出不支援 P3 廣色域的螢幕、TouchID 也是從 2018 年以後全面成為標配、SSD 和 Ram 的速度被閹割、明明都跟 Macbook Air 一樣重了但電池卻更小、都 2026 年了怎麼還在用 USB2.0⋯⋯等等缺點
每個缺點在目前其他 Mac 上都不存在，Neo 真的是來拖後腿的
但 Neo 勝在它便宜啊，不跟 Mac 其他產品線來比的話它完全碾壓其他同價位的新品

如果我們現在打開 Amazon，可以發現同價位的筆電基本上沒辦法在這幾點上贏過 Macbook Neo：
1. 螢幕
螢幕基本上就已經打趴所有同價位產品了，甚至連 800 鎂的輕薄筆電都不一定有 2k sRGB 螢幕
2. 全鋁機殼
基本上都是塑膠機殼
3. 好用的觸控板
別說同價位了，我就還沒有在其他廠牌的筆電上用過跟 Macbook 一樣好用的觸控板
4. 超長續航
無論是最新一代的 x86 或 Arm，續航都還是沒辦法跟 Macbook 一決高下

同時這也是開拓一片藍海的機會，像是有些只是需要便宜的筆電應付一些日常休閒使用、但又不想用平板的客群

至於功能的部分基礎的文書、影音一定沒問題，甚至不如說 Macbook Neo 就是為了這些使用場景設計的
> 唯一需要注意的可能只有用 Chromium 瀏覽器的時候會光速把 Ram 燒掉，但 Safari 應該用起來是輕輕鬆鬆

在影視颶風的測試中，他們同時疊兩支 4k 影片回放相當流暢，要輕度剪輯應該也沒毛病，或是先輸出 FHD 的代理檔剪輯

> 真的要說的話，Macbook Neo 的競爭對手，應該只有二手的 M1/M2 Macbook Air 了吧

## Macbook Neo 適合哪些人？

這邊結合 MKBHD 和我自己的看法，整理成如下

### 中小學生

絕對沒問題，只是要做學校報告、看影片、查資料、學習等這些不吃效能的操作，絕對符合

### 大學生

看科系，如果只是要拿來做一些和 [中小學生](#%E4%B8%AD%E5%B0%8F%E5%AD%B8%E7%94%9F) 一樣的使用情境，也是沒問題的
不過如果是需要用到一些專業軟體、或是跟我一樣是臭寫 code 的，我還是不太推薦
像我現在平常固定開的幾個 App 就會噴掉 10GB 了，還不包含我測試時可能需要開的 VM 或開發環境
還有之前試著用 GitHub Copilot Agent 寫 iOS app 的時候，16GB 的 Ram 甚至不太夠用，開 iOS 模擬器的時候操作都有些頓頓的，我上次有這種感覺已經是還在用 Intel Mac 時了

### 攝影師

修圖軟體我覺得在 Macbook Neo 上可用，但不是很好用
像是＿＿＿在影片中一次導入多張照片並編輯，就能感覺到明顯的不順暢
不過既然這是吃飯工具，而且專業相機和鏡頭也是動輒幾萬起跳，一台 Macbook Air/Pro 應該不算是太重的開銷才對

### 軟體工程師

寫程式方面我會同時說 `很適合` & `很不適合`

⋯⋯對，我知道我前面才說不推薦，不過我是有理由的（？

理由是因為軟體工程師的範圍太廣了，像是我幾乎不開發 app 而且目前只有寫輕量前後端，**可能** 用 Macbook Neo 還跑的動
不過像是開發 app 這種時常會掛模擬器的肯定跑不動，或是有些大型軟體開發沒意外也是跑不動
既然目前得出的結果都是不適合，我怎麼還會說 `很適合` 呢？

因為有些軟體開發流程是 **完全** 不在本地端編譯 & 執行的，本地端只要能寫 code 能開整合開發環境（通常 loading 都不會很重）、能連線上伺服器就好了
甚至聽過有直接在伺服器端寫程式的，只要有個能連線的裝置即便是 iPad 都能用來工作

不過我覺得軟體工程師應該是知道自己 *需要* 和 *不需要* 哪些需求的，應該也不會這麼煩惱才對

### 影片剪輯

我個人認為不適合，不過如果只是要剪 shorts 之類的這種手機早就能剪的東西，Macbook Neo 一定也沒問題
不過如果像是影視颶風或是極客灣這種有大量特效、調色的重活，我覺得即便不是像他們都剪 4k 或是轉成 FHD 的代理檔，應該還是跑不動
我覺得如果是剪輯師的話應該還是乖乖直上 Macbook Pro 吧

## 結語

就像是 MKBHD 說的

> ... like if you're watching a tech video on YouTube, if you're subscribed to this channel, this probably isn't meant for you.
> *如果你有在 yt 上看科技相關影片，或是你有訂閱這個頻道，這（Macbook Neo）應該對你不重要*

對於重度筆電使用者的我（會電量焦慮、抱怨續航連六小時都撐不到、注重效能）來說，Macbook Neo 真的不重要
入門款筆電、用手機晶片、基本上是設計給基礎功能（文書、網頁瀏覽）、螢幕只有 60Hz sRGB，每一點對我來說都是缺點
但勝在他便宜啊，在同價位沒有競爭對手，**會用這個價格買筆電的人不會在乎這些缺點、會在乎這些缺點的人不會買這台筆電**
只能說感嘆蘋果精準的刀法，一刀切在這巧妙的位置
而且我相信蘋果這精準的刀法會為他們帶來不錯的銷量和利潤

## 外傳：精準的蘋果刀法

忘記之前是看極客灣的雲飛還是誰說的（我印象中是雲飛），蘋果跟其他廠商的定價邏輯——或是說做產品的邏輯——不一樣
別的廠商基本上都是先訂出 **目標規格** 再來找出對應的客群能接受的價格，最後才是在成本限制內找零件組合產品
像是如果現在想要設計出一台新筆電，希望是 i5 + 4060 + 16GB Ram + 512GB SSD
根據這個 spec 對應的目標客群大概能接受一台筆電在 45k 上下，其中成本必須抓在 30k 內（這邊金額是假設的，我其實不知道現在遊戲筆電大概的價位在多少 & 成本是多少
最後採購想辦法跟各個廠商談出符合成本的價格

而蘋果的邏輯則是，先訂出 **目標客群** 再來找出這個客群需要的規格，最後才是訂出一個目標客群能接受的價格
同時蘋果極高的銷售（如果按照產品型號來看，現在應該很少有單型號能賣的比蘋果高的廠商）保證了就算利潤相較之下比較差，其他廠商們都還是會接單
再來是蘋果大幅降低零件數量，同一代晶片就那些選擇（雖然基礎款、Pro、Max 再加上各種容量的 Ram 排列組合也很多種，但應該還是比 x86 平台的組合來的少）、其他硬體零件基本上都沒有改（螢幕、鍵盤等），相較於其他廠商有各式各樣的產品外觀有庫存壓力，蘋果可能會想說：`喔外殼下單太多了，那留著明年用吧`

還有蘋果為了節省成本直接採用 A18 Pro 當作 Macbook Neo 的晶片也蠻聰明的，我認為 Macbook Neo 應該跟 iPhone 16 開發週期是重疊的——或是至少是在同一年度開始的
算一算發現 A18 Pro 拿來給 Macbook Neo 用剛好，那就直接用吧，啊頻寬不夠那就遮蔽一顆 GPU 就好反正開發成本早就回收了
還有新一代 Studio Display 也是，據說 Studio Display XDR 裡面直接塞了顆有 12GB Ram 的 A19 Pro，以支援複雜的 mini-LED 背光計算

> 是的沒錯，一台螢幕用的晶片比入門款筆電還高階，想不到吧
> 雖然他們價格差了五倍啦
> 所以我說蘋果要不要更新一下驅動，讓 Studio Display XDR 可以跑降畫質的 macOS XD

換作其他廠商應該會叫晶片部門再另外生一顆主控出來，不過蘋果的邏輯是 `這東西能用 & 沒爆預算，那就直接用吧想啥`

又双叒一次讚嘆蘋果的——或是說庫克的——精準刀法，真不愧是供應鏈出身的 CEO
